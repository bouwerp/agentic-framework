# Installation Guide

## Quick Install

```bash
# Clone the repository
git clone https://github.com/bouwerp/agentic-framework.git
cd agentic-framework

# Run the installer (auto-detects platform)
./scripts/install.sh

# Verify installation
./scripts/verify.sh
```

## What Gets Installed

The installer detects your platform and installs the appropriate components:

### Skills (4) — All Platforms
- `figma-interaction/` - Figma integration via MCP or REST API
- `jira/` - JIRA interaction with ADF support
- `confluence/` - Confluence interaction with ADF support
- `github/` - GitHub PR workflow via `gh` CLI

### Tools (3) — OpenCode Only
- `gsr.ts` - Global Search & Replace
- `figma-rest.ts` - Figma REST API integration
- `figma-oauth.ts` - Figma OAuth authentication

### Tools (1) — Gemini Only
- `gsr.sh` - Shell-based Global Search & Replace

### Platform-Specific Components

| Component | OpenCode | Claude Code | Gemini | Cursor | Pi |
|-----------|----------|-------------|--------|--------|-----|
| **Skills** | All 4 | All 4 | All 4 | All 4 | All 4 |
| **TypeScript tools** | All 3 | — | — | — | — |
| **Shell tools** | — | — | gsr.sh | — | — |
| **MCP setup** | — | Instructions | — | Instructions | — |
| **Config** | opencode.json | — | — | — | — |

## Manual Installation

If you prefer to install manually for your specific platform:

### OpenCode
```bash
cp -r .opencode/tools ~/.config/opencode/
cp -r .opencode/skills ~/.config/opencode/
cp opencode.json ~/.config/opencode/  # optional
```

### Claude Code
```bash
mkdir -p ~/.claude/skills
cp -r .opencode/skills/* ~/.claude/skills/

# Optional: set up MCP
claude mcp add --transport http figma https://mcp.figma.com/mcp
claude mcp auth figma

# Optional: install plugins
claude plugin install figma@claude-plugins-official
claude plugin install jira@claude-plugins-official
claude plugin install confluence@claude-plugins-official
```

### Gemini
```bash
mkdir -p ~/.gemini/skills ~/.gemini/tools
cp -r .opencode/skills/* ~/.gemini/skills/
# gsr.sh is generated by install.sh
```

### Cursor
```bash
mkdir -p ~/.cursor/skills
cp -r .opencode/skills/* ~/.cursor/skills/

# In Cursor chat:
/plugin-add figma
/plugin-add jira
```

### Pi
```bash
mkdir -p ~/.pi/agent/skills
cp -r .opencode/skills/* ~/.pi/agent/skills/
```

Note: Pi uses skills as capability packages with CLI tools, no MCP support.
See https://shittycodingagent.ai

## Environment Setup

Add these to your `~/.zshrc` or `~/.bashrc`:

```bash
# Figma Personal Access Token
# Get from: https://www.figma.com/developers/api#access-tokens
export FIGMA_PERSONAL_TOKEN='figd_your-token-here'

# Atlassian API Token (for JIRA/Confluence)
# Get from: https://id.atlassian.com/manage-profile/security/api-tokens
export ATLASSIAN_API_TOKEN='your-token-here'
export ATLASSIAN_EMAIL='your.email@example.com'
export ATLASSIAN_SITE='your-domain.atlassian.net'
```

## Verification

After installation, run:

```bash
./scripts/verify.sh
```

This checks:
- ✓ Platform detected
- ✓ All tools are installed (platform-specific)
- ✓ All skills are installed
- ✓ Environment variables are set
- ✓ Configuration is valid

## Testing

Test the framework with:

```bash
# OpenCode
opencode run "Get design tokens from Figma file YOUR_FILE_KEY"
opencode run "Rename all instances of 'foo' to 'bar' in *.js files"

# Claude Code
claude "Extract colors and spacing from this Figma design: <url>"

# Gemini
gemini "Get variables from Figma file ABC123"

# Cursor
# Open agent chat and ask about a Figma design or JIRA issue

# Pi
pi -p "Get design tokens from Figma file YOUR_FILE_KEY"
```

## Troubleshooting

### Skills not showing up
Restart your tool or reload your shell:
```bash
source ~/.zshrc  # or source ~/.bashrc
```

### Tools not available (OpenCode)
Verify tools are in the correct location:
```bash
ls ~/.config/opencode/tools/
# Should show: figma-oauth.ts  figma-rest.ts  gsr.ts
```

### Permission errors
Make scripts executable:
```bash
chmod +x scripts/install.sh scripts/verify.sh
```
